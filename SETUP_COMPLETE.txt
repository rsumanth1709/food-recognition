🏗️  ARCHITECTURE:
════════════════════════════════════════════════════════════════════════════════

Model:
  Input (224×224×3 RGB image)
       ↓
  Data Augmentation (flip, rotate, zoom)
       ↓
  EfficientNetB3 (pretrained on ImageNet)
       ↓
  Global Average Pooling
       ↓
  Dense layers with dropout and batch norm
       ↓
  ┌─────────────────────┬─────────────────────┐
  │ Classification Head │   Regression Head   │
  │ (101 classes)       │  (Calorie estimate) │
  │ Softmax output      │    Linear output    │
  └─────────────────────┴─────────────────────┘
       ↓                            ↓
  Food class + confidence    Calorie prediction

⚠️  IMPORTANT NOTES:
════════════════════════════════════════════════════════════════════════════════

1. DATASET REQUIRED
   The Food-101 dataset is required for training.
   Download from: https://www.kaggle.com/dansbecker/food-101
   Extract to: data/food-101/

2. PRETRAINED WEIGHTS
   The model uses EfficientNetB3 pretrained on ImageNet.
   First run will download weights (~150MB).

3. GPU OPTIONAL
   Training works on CPU but is 10-20x slower.
   GPU highly recommended for practical use.

4. PORTS
   Streamlit uses port 8501
   Flask uses port 5000
   Make sure these are available.
